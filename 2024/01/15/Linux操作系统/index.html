Operating System

Operating-system Structure

两个接口：

  • users:
    • CLI 通过终端向操作系统发布指令
    • GUI 图形化
    • Batch 批处理
  • apps:
    • API -> system call(OS level code -> kernel mode)
    • trap mechamism -> dual mode(user mode / kernel mode)

dual mode（用一个硬件模式位来表示当前模式位）：

  • user mode：用户级的代码
  • kernel mode：操作系统级的代码

Process Concept

进程的定义：

  • 进程时一个程序的一次执行过程：

    • 能完成具体的功能（一系列指令）
    • 是在某个数据集合上完成的（data，堆栈）
    • 执行过程是可并发的

  • 进程是资源分配、保护和调度的基本单位

  • 程序：被动的，一个包含指令的存在磁盘上的可执行文件

  • 进程：活动的，当程序被加载进内存，就是进程（包含一个PC）

Process in Memory:
回忆在计组课程中写MIPS代码时，就区分了data区和text区

text区域存放指令（read only）
data区域直到程序结束才会回收

并发Concurrency：多个事件同时发生或存在（单核CPU）

并行：同时运行

Process State：

  • Running运行态：此时进程的代码在CPU上运行（单核CPU系统中只能有一个进程处于运行态）
  • Ready就绪态：进程具备运行条件，等待分配CPU
  • Waiting等待态：进程在等待某些事件的发生（比如IO操作结束）

中断技术

系统调用时中断的一个特例

中断源：

  • 外中断：
    • 处理器之外的信号
    • 外部中断均是异步中断
  • 内中断（异常Exception）：
    • 处理器内部的信号
    • 硬件异常：掉电，奇偶检验
    • 程序异常：越界，除0
    • 系统调用

特权指令：只能运行在内核模式下运行的指令
非特权指令：只能运行在用户模式下运行的指令

模式切换：
中断时用户态向核心态转换的唯一途径。

进程切换：

  • 切换时机：
    • 主动进入等待状态
    • 被动进入就绪状态
  • 切换过程
    • 保存中断进程的上下文信息
    • 保存被中断进程的控制信息
    • 将被中断的进程加入相应的序列
    • 调度一个新的进程并恢复它的上下文信息

进程控制块（状态）PCB

进程队列：只把PCB放入队列
